# 新手引导页面问题排查指南

## 问题现象
点击"继续"或"跳过"按钮没有反应

## 已修复的问题

### 1. 导航方式错误
**问题：** `app.js` 使用 `wx.reLaunch()` 打开引导页，导致引导页成为栈中唯一页面，`navigateBack()` 无法工作

**修复：** 将 `guide.js` 中的导航方式改为 `wx.redirectTo()`

```javascript
// 修复前
wx.navigateBack()

// 修复后
wx.redirectTo({ url: '/pages/index/index' })
```

### 2. WXML 事件绑定语法
**问题：** `bind:tap` 应该改为 `bindtap`（两种语法都支持，但 `bindtap` 更标准）

**修复：** 已更新为 `bindtap`

### 3. 缺少 hover 反馈
**问题：** 按钮没有点击反馈，用户不知道是否点击成功

**修复：** 添加了 `hover-class` 样式

## 测试步骤

### 清除缓存重新测试
1. 打开微信开发者工具
2. 点击工具栏 "清除缓存" → "清除全部缓存"
3. 勾选 "清除数据缓存"
4. 重新编译项目

### 查看控制台日志
点击按钮后应该看到以下日志：
```
onNext called, currentStep: 0  // 或 onSkip called
completeGuide called
redirectTo success
```

### 如果仍然没有反应
1. 检查控制台是否有错误信息
2. 确认 `hasGuidance` 缓存值（可能已经设置为 true）
3. 在微信开发者工具中，点击 "清除缓存" → "清除全部缓存"

## 手动测试 hasGuidance
在控制台输入：
```javascript
wx.getStorageSync('hasGuidance')
```
如果返回 `true`，说明引导页已经显示过，需要清除缓存才能再次看到引导页

## 强制重置引导状态
在控制台输入：
```javascript
wx.removeStorageSync('hasGuidance')
```
然后重新编译项目

## 常见问题

### Q: 点击按钮后页面没有跳转
A: 检查控制台是否有 `redirectTo success` 日志，如果有但页面没变化，可能是目标页面路径错误

### Q: 根本没有看到引导页
A: 检查 `hasGuidance` 是否已经为 `true`，如果是请清除缓存

### Q: 按钮点击没有反应
A: 检查控制台是否有 `onNext called` 或 `onSkip called` 日志，如果没有说明事件绑定有问题
