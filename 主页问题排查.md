# 主页面问题排查指南

## 问题现象
主页面打不开

## 已修复的问题

### 1. 循环依赖问题
**问题：** `utils/db.js` 引用 `utils/error.js`，可能导致循环依赖

**修复：** 移除 `db.js` 对 `error.js` 的依赖，使用内联错误处理

### 2. 引导页跳转逻辑
**问题：** `hasGuidance=true` 时没有显式跳转到首页

**修复：** 在 `app.js` 中添加显式的 `wx.switchTab` 跳转到首页

---

## 排查步骤

### 1. 清除缓存
在微信开发者工具中：
1. 点击菜单 "工具" → "清除缓存" → "清除全部缓存"
2. 勾选 "清除数据缓存"
3. 重新编译项目

### 2. 查看控制台日志
启动后应该看到以下日志：
```
App onLaunch
settings: { dailyCount: 20, playMode: 'manual', playSpeed: 1.0 }
hasGuidance: false (或 true)
```

如果 `hasGuidance: false`：
```
首次启动，显示引导页
```

如果 `hasGuidance: true`：
```
已显示过引导，跳转到首页
```

### 3. 强制重置引导状态
如果引导页有问题，在控制台执行：
```javascript
wx.removeStorageSync('hasGuidance')
```
然后重新编译项目

### 4. 检查页面路径
确认 `app.json` 中页面路径正确：
```json
{
  "pages": [
    "pages/guide/guide",
    "pages/index/index",
    ...
  ]
}
```

---

## 常见错误及解决方案

### 错误 1: `require` 失败
**错误信息：** `require utils/error.js` 失败

**解决方案：** 已修复，`db.js` 不再引用 `error.js`

### 错误 2: 页面不存在
**错误信息：** `page not found`

**解决方案：** 
1. 检查文件是否存在
2. 检查 `app.json` 中路径是否正确

### 错误 3: 云函数调用失败
**错误信息：** `cloud.callFunction fail`

**解决方案：**
1. 确认云开发环境已初始化
2. 确认云函数已部署

## 手动测试首页加载

在控制台执行：
```javascript
// 直接跳转到首页
wx.switchTab({ url: '/pages/index/index' })

// 检查当前页面
console.log(getCurrentPages().map(p => p.route))
```

