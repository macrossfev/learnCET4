# 四级必过 - 微信小程序CET-4词汇速记工具 开发文档

## 1. 项目概述

**小程序名称：** 四级必过
**小程序简称：** 必过
**小程序介绍：** 3393个四级真题高频词，按20年真题词频排序，最常考的排最前。每天20词，美音朗读，学完即测，间隔复习。不背废词，只背必考词，一次过四级。

基于微信小程序的CET-4词汇速记工具，以历年真题词频为核心排序依据，只收录最高频的1000词，帮助考生用最短时间拿下四级词汇。通过广告变现。

**当前版本范围：** CET-4 前1000高频词
**后续扩展：** 全量3393词、CET-6、考研词汇等

## 4. 词频分析流程

### 4.1 数据采集

```
来源：CET-4历年真题文本（1987年至今）
内容：阅读理解、完形填空、翻译题、听力原文
格式：整理为纯文本文件，按年份命名
存放：/data/raw_texts/cet4/
```

### 4.2 处理流程

```
原始真题文本
    ↓
Python + spaCy 分词
    ↓
词形还原（lemmatization）：goes→go, abandoned→abandon
    ↓
去除停用词（the, a, is, are 等）
    ↓
词频统计（Counter）
    ↓
与CET-4大纲词表取交集
    ↓
按频次降序排列
    ↓
输出带频次排名的词表
```

### 4.3 关键注意事项

- 义项选择：取该词在真题中最高频的义项，非词典默认第一义项
  - 例：address 真题高频义为"解决/处理"，非"地址"
  - 例：issue 真题高频义为"问题"，非"发行"
- 词组选择：从真题原文中提取该词实际出现的搭配
- 例句：控制在10词以内，优先从真题中改编简化

### 4.4 词频分布分析

基于20年CET-4真题文本统计，3393个有效词条的频次分布如下：

| 排名 | 词数 | 每词出现频次 | 累计覆盖率 | 说明 |
|------|------|-------------|-----------|------|
| 1-100 | 100词 | 84-506次，平均141次 | ~23% | 超高频核心词 |
| 101-300 | 200词 | 46-84次，平均62次 | ~40% | 高频词 |
| 301-500 | 500词 | 32-46次，平均37次 | **56%** | 前500词过半覆盖 |
| 501-1000 | 500词 | 17-32次，平均23次 | **75%** | 前1000词覆盖3/4 |
| 1001-2000 | 1000词 | 6-17次，平均11次 | ~90% | 中低频 |
| 2001-3393 | 1393词 | 1-6次，平均3次 | 100% | 低频长尾 |

**关键结论：前1000词覆盖真题75%词汇量，性价比最高。**

### 4.6 学习周期预估

基于1000词、复习间隔1/3/7/14/30天的模型：

| 每日学习量 | 学完天数 | 含复习总周期 | 每日耗时 | 适用场景 |
|-----------|---------|-------------|---------|---------|
| 10词/天 | 100天 | ~130天 | 10-15分钟 | 长线备考 |
| **20词/天（默认）** | **50天** | **~80天** | **15-25分钟** | **常规备考（推荐）** |
| 30词/天 | 34天 | ~65天 | 25-35分钟 | 中期冲刺 |
| 50词/天 | 20天 | ~50天 | 40-60分钟 | 临考突击 |

说明：
- 每日耗时包含新词学习（6段音频 + 阅读）+ 复习 + 测试
- 复习量中期会累积，高峰期每天约有新词 + 30-50个复习词
- 已认识的词可直接标记"已掌握"跳过，实际进度因人而异
- 产品宣传口径：**每天15分钟，30天突破四级词汇**

## 5. 音频生成流程

### 5.1 生成内容

每个词条生成6个音频文件：

| 序号 | 内容 | 语言 | 语音 |
|------|------|------|------|
| 1 | 单词（如 abandon） | 英文 | en-US-JennyNeural（美音女声） |
| 2 | 中文释义（如 放弃） | 中文 | zh-CN-XiaoxiaoNeural |
| 3 | 词组（如 abandon the plan） | 英文 | en-US-JennyNeural |
| 4 | 词组中文（如 放弃计划） | 中文 | zh-CN-XiaoxiaoNeural |
| 5 | 例句（如 He abandoned his dream.） | 英文 | en-US-JennyNeural |
| 6 | 例句中文（如 他放弃了梦想。） | 中文 | zh-CN-XiaoxiaoNeural |

### 5.2 批量生成脚本

使用 edge-tts Python库批量生成：

```bash
pip install edge-tts
```

```python
import edge_tts, asyncio, json

async def generate_audio(text, voice, output_path):
    communicate = edge_tts.Communicate(text, voice)
    await communicate.save(output_path)

async def batch_generate(words_json_path):
    with open(words_json_path) as f:
        words = json.load(f)
    for w in words:
        await generate_audio(w["word"], "en-US-JennyNeural", f"audio/words/{w['word']}.mp3")
        await generate_audio(w["meaning"], "zh-CN-XiaoxiaoNeural", f"audio/meanings/{w['word']}_meaning.mp3")
        await generate_audio(w["phrase"], "en-US-JennyNeural", f"audio/phrases/{w['word']}_phrase.mp3")
        await generate_audio(w["phrase_meaning"], "zh-CN-XiaoxiaoNeural", f"audio/phrase_meanings/{w['word']}_phrase_meaning.mp3")
        await generate_audio(w["example"], "en-US-JennyNeural", f"audio/examples/{w['word']}_example.mp3")
        await generate_audio(w["example_meaning"], "zh-CN-XiaoxiaoNeural", f"audio/example_meanings/{w['word']}_example_meaning.mp3")

asyncio.run(batch_generate("data/words_cet4.json"))
```

### 5.3 存储与缓存策略

- 音频文件上传至云开发云存储
- 小程序端使用 `wx.downloadFile` + `wx.getFileSystemManager` 缓存已播放音频
- 按学习单元预加载：进入某单元时，后台下载该单元全部音频
- 缓存上限管理：保留最近5个单元的缓存，自动清理更早的

## 10. 关键技术要点备忘

### 音频播放队列实现

单词卡片的6段音频需顺序播放，核心逻辑：

```javascript
// utils/audio.js
const playSequence = (audioList) => {
  let index = 0;
  const ctx = wx.createInnerAudioContext();

  const playNext = () => {
    if (index >= audioList.length) {
      // 序列播放完毕
      if (isContinuousMode) {
        // 连续模式：触发翻页并播放下一词
        triggerNextCard();
      }
      return;
    }
    ctx.src = audioList[index];
    ctx.play();
    index++;
  };

  ctx.onEnded(playNext);
  ctx.onError((err) => {
    console.error('音频播放失败', err);
    index++;
    playNext(); // 跳过错误音频继续播放
  });

  playNext();
};
```

### 干扰项生成逻辑

选择题的干扰项应避免过于简单或过于相似：

```javascript
// 从同频段（±50排名）中随机选3个不同词的释义
```

### 默写判定逻辑

```javascript
// 不区分大小写
// 去除首尾空格
// 比对每个字母，标记错误位置用于红色高亮
const checkSpelling = (input, correct) => {
  const a = input.trim().toLowerCase();
  const b = correct.toLowerCase();
  if (a === b) return { correct: true };
  // 逐字符比对，返回差异位置
  const diff = [];
  for (let i = 0; i < Math.max(a.length, b.length); i++) {
    if (a[i] !== b[i]) diff.push(i);
  }
  return { correct: false, diff, expected: b };
};
```
